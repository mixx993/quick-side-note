# Quick Side Note v1.6.0 单词学习包方案

## 产品目标

把“单词本”里长期积累的零散单词整理成一份可以直接阅读和复习的知识成果，同时保持现有侧键便签操作不变。

## 用户流程

1. 用户继续通过侧键 OCR、翻译或手动输入积累单词。
2. 点击主窗口“整理”、托盘菜单“整理单词”或按 `Ctrl+M`。
3. 选择“最近 30 个”或“全部”。
4. 点击“生成学习包”。
5. 在结果区继续编辑、保存修改、复制全部或打开成果目录。

## 数据来源

- 命名中包含“单词”的便签页；若没有该名称，则使用当前页。
- `vocabulary.jsonl` 中已有的 OCR 翻译记录。
- 支持 `原文：...` / `译文：...` 成对格式，以及单独一行的英文单词或短语。
- 按不区分大小写的英文单词去重，优先保留已有中文释义。

原有 `note*.txt` 和 `vocabulary.jsonl` 不迁移、不改写。生成结果单独保存到：

```text
%USERPROFILE%\Documents\QuickSideNote\studio\vocabulary-study-pack-*.md
```

## 学习包结构

- 主题分组
- 易混词与常用搭配
- 例句与中文翻译
- 记忆路线
- 复习小测与答案

## 交互边界

- 不引入捕获记录、知识图谱、标签系统或向量数据库。
- 不要求用户逐条维护元数据。
- API 生成在后台线程执行，不阻塞便签窗口。
- 关闭生成窗口后，迟到结果会被丢弃，不会写入成果目录。
- 只有用户点击“生成学习包”后，所选单词和已有释义才会发送给 DeepSeek。

## 验收标准

- 默认窗口和 540×440 最小窗口下，生成与结果操作按钮始终可见。
- 当前单词数据可以正确识别、去重，并支持最近 30 个与全部范围。
- 生成失败时窗口保持可操作，可以直接重试。
- 成果自动保存为 Markdown，编辑后可原位保存。
- 原便签、翻译记录、侧键与 OCR 流程无回归。
