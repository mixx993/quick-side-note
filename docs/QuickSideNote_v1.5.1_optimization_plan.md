## 0. 实施记录

2026-07-10 已完成 v1.5.1 范围内的 S1、S2、H1、T1、P1、I1、B1、W1 和 Q1：

- 便签和状态文件使用原子写入，并保留 `.bak` 备份与启动恢复路径。
- 编辑器使用 600 ms 防抖自动保存，状态栏显示真实保存状态。
- 鼠标与键盘钩子失败后立即退出线程并采用退避重试；钩子回调不再同步写日志。
- OCR/翻译结果通过主线程事件队列回收；框选或翻译中再次按侧键可以取消任务。
- 图片兜底默认关闭并要求设置页明确授权；日志不再记录 OCR 原文并支持轮转。
- 卸载时删除本应用固定名称的开机启动项，保留便签、生词本和 API Key。
- 构建脚本校验版本和外部工具退出码，已生成安装器、便携 ZIP 与 SHA-256。
- 补充原子保存、恢复、隐私、钩子失败、翻译任务、托盘菜单、多屏和打包脚本测试；当前共 54 项单元测试通过。

本次未执行真实覆盖安装或卸载，以避免影响当前正在使用的 v1.5.0 安装和用户数据。安装器已由 Inno Setup 成功编译；已启动的旧版本实例也验证了新包的单实例保护。代码签名和安全凭据迁移属于 v1.6.0，需要证书与迁移窗口后再实施。

## 1. 背景与结论

v1.5.0 已完成 UI 更新、托盘入口、首次配置、安装器和 GitHub Release，现有单元测试全部通过。当前没有必须撤回版本的 P0 问题，但在数据保存、Windows 钩子生命周期、翻译任务取消、隐私披露、卸载清理和构建可靠性方面仍有明显风险。

下一阶段不应优先扩展 Studio、思维导图或报告等新功能。建议先发布 v1.5.1 稳定性版本，消除可能造成便签丢失、后台线程堆积、截图意外上传和错误安装包发布的问题；再在 v1.6.0 处理中长期架构与凭据安全。

## 2. 优化目标

v1.5.1 需要达到以下结果：

- 保存过程即使被中断，也不会破坏已有便签和设置。
- 输入后能够真实自动保存，并准确显示保存状态。
- 鼠标与键盘钩子安装失败时不会持续创建后台线程。
- 翻译任务可以取消，应用退出后不会再调用已销毁的 Tk 窗口。
- 默认不会在用户不知情时上传原始截图。
- 日志不再长期保存完整 OCR 原文。
- 卸载后不会遗留失效的开机启动项。
- 构建工具失败时，脚本必须失败退出，不能误用旧产物。
- 高风险 Windows 路径具有可重复验证的测试。

v1.6.0 的目标是降低后续维护成本：

- API Key 使用 Windows 安全凭据存储。
- Win32、存储、翻译任务和 Tk UI 分离为清晰模块。
- 建立打包应用的 Windows 集成测试和签名发布流程。

## 4. 详细实施方案

### 4.1 S1：原子保存与恢复

新增统一的安全写入函数，所有便签和 `state.json` 都通过该函数落盘：

1. 在目标文件同目录创建临时文件。
2. 写入完整内容并执行 `flush()`。
3. 在需要时执行 `os.fsync()`，确保内容交给操作系统。
4. 将现有文件保留为 `.bak`，最多保留一份最近成功版本。
5. 使用 `os.replace()` 原子替换目标文件。
6. 任何步骤失败都保留原文件，并向用户显示可理解的错误。

启动时读取 `state.json` 的恢复顺序：

1. 读取主文件。
2. 主文件损坏时读取 `.bak`。
3. 两者都失败时扫描现有 `note*.txt` 重建页签列表。
4. 将恢复结果写入日志，但不记录便签内容。

验收标准：

- 模拟写入异常后，旧便签仍可正常打开。
- 损坏 `state.json` 后，自定义页签对应的 txt 文件不会失去入口。
- 磁盘写入失败会提示用户，不会显示“已保存”。

### 4.2 S2：真实自动保存

监听文本框 `<<Modified>>` 事件，并使用 500 毫秒防抖：

- 输入发生后立即显示“未保存”。
- 防抖计时期间继续输入时重置计时器。
- 开始写入时显示“保存中”。
- 保存成功后显示“已保存”。
- 保存失败时显示“保存失败”，保留未保存状态并允许重试。

切页、隐藏、退出、启动 OCR 和窗口失焦时仍执行一次立即保存，作为防抖保存的补充。

验收标准：

- 连续输入不会每个字符都写磁盘。
- 停止输入 500 至 800 毫秒后文件内容更新。
- 退出前存在待保存任务时，必须先执行最终保存。
- UI 状态与实际文件状态一致。

### 4.3 H1：钩子生命周期

鼠标钩子和键盘钩子采用相同的生命周期规则：

- `SetWindowsHookExW` 失败后设置错误状态并立即退出线程，不进入消息循环。
- 每个线程只维护自己的 `thread_id` 和 `hook_id`，新重试不能覆盖仍存活线程的状态。
- 健康检查使用指数退避，例如 1、2、4、8、30 秒，避免持续重试。
- 达到最大连续失败次数后停止自动重试，并在托盘或设置页提示用户。
- 停止应用时发送 `WM_QUIT`，等待线程退出并验证 `UnhookWindowsHookEx` 已执行。
- 低级钩子回调只做常量判断和队列写入，不执行文件日志写入。

验收标准：

- 模拟钩子安装失败 10 分钟，线程数不增长。
- 钩子线程意外结束后可以恢复一次正常监听。
- 退出应用后不存在 Quick Side Note 钩子线程。

### 4.4 T1：翻译任务取消与线程边界

建立统一的翻译任务对象，至少包含：

- `task_id`
- 当前阶段：框选、OCR、翻译、完成、失败、取消
- 创建时间和总截止时间
- 取消标记
- 临时截图路径

线程规则：

- 后台线程不得调用任何 Tk 方法，包括 `root.after()`。
- 后台线程只把结果写入现有事件队列。
- 主线程的 `_poll_events()` 负责更新 UI、追加便签和清理状态。
- 应用退出后将状态设为 closing，忽略迟到结果。
- 新任务启动时使旧任务结果失效，防止重复写入。

取消规则：

- 框选阶段再次按侧键或 `Esc`：立即关闭框选层。
- OCR 或网络翻译阶段再次按侧键：标记任务取消，恢复可操作状态。
- 底层网络调用无法立即终止时，返回结果也不得写入便签。
- 网络请求设置连接、读取和总任务截止时间；超时后允许重试一次。

验收标准：

- 翻译中再次按侧键，界面在 200 毫秒内恢复可操作状态。
- 取消任务完成后不会把迟到结果追加到便签或生词本。
- 翻译中退出应用不会出现 Tk 跨线程异常。
- 连续触发两次任务只接受最新有效任务结果。

### 4.5 P1：隐私与日志

默认策略：

- Windows OCR 在本机处理图片。
- DeepSeek 只接收 OCR 后的文本。
- Windows OCR 失败时，不自动把截图交给 Codex CLI。
- 如保留图片兜底，必须在设置页作为默认关闭的独立选项，并说明服务商、上传内容和用途。

日志调整：

- 不记录完整 OCR 原文、API Key、请求头、截图路径或完整服务端响应。
- 正常日志只记录任务编号、文本长度、阶段耗时、错误类型和 HTTP 状态码。
- 日志达到 1 MB 后轮转，最多保留 3 份。
- 设置页增加“打开日志目录”和“清除诊断日志”。

验收标准：

- 搜索日志文件无法找到测试 OCR 原文或 API Key。
- OCR 失败时不会在未授权情况下调用图片模型。
- 隐私说明与程序真实数据流一致。

### 4.6 I1：安装、升级与卸载

安装器继续使用固定 `AppId`，使新版本覆盖升级旧版本。新增卸载清理：

- 删除 `HKCU\Software\Microsoft\Windows\CurrentVersion\Run` 下名为 `QuickSideNote` 的值。
- 不删除整个 Run 键。
- 默认保留 `%USERPROFILE%\Documents\QuickSideNote`。
- 默认保留 `DEEPSEEK_API_KEY`，并在卸载说明中告知如何手动清除。
- 升级安装前关闭运行中的 `QuickSideNote.exe`，安装后只启动一个实例。

验收标准：

- 启用开机启动后卸载，Run 值被清除。
- 卸载后便签和生词本仍存在。
- 覆盖安装不会重复创建桌面或开始菜单快捷方式。
- 从 v1.5.0 升级后原数据和设置仍可读取。

### 4.7 B1：可靠构建

构建脚本需要执行以下检查：

- PyInstaller 结束后立即检查 `$LASTEXITCODE`，非零则退出。
- 构建前删除当前目标版本的旧产物，避免复用残留文件。
- 验证 `dist\QuickSideNote.exe` 的修改时间和内部版本。
- Inno Setup 结束后立即检查 `$LASTEXITCODE`，非零则退出。
- 只检查当前目标版本的精确安装器文件名，不使用“目录中最新的安装器”。
- 输出 EXE 和 ZIP 的 SHA-256。
- 构建开始前验证应用、安装器和文档版本号一致。

验收标准：

- 人为让 PyInstaller 失败时，脚本返回非零且不产生成功提示。
- 人为让 Inno Setup 失败时，不会选中旧安装包。
- 构建结果中的版本号、文件名和文档版本一致。

### 4.8 W1：Windows 兼容性

副屏定位：

- 使用 `MonitorFromPoint` 和 `GetMonitorInfoW` 获取鼠标所在显示器工作区。
- 支持副屏负坐标、上下排列和不同 DPI。
- 便签始终完整显示在目标显示器工作区内。

便签读取：

- 捕获 `OSError` 和 `UnicodeDecodeError`。
- 优先读取 UTF-8 和 UTF-8 BOM。
- 无法解码时保留原文件，提供备份副本和明确提示。
- 不静默覆盖无法读取的文件。

验收标准：

- 在左侧负坐标副屏触发时，便签出现在鼠标附近。
- ANSI 或损坏文件不会导致无控制台版本静默退出。

## 8. 兼容与迁移策略

- v1.5.1 不修改现有 txt 和 JSONL 数据格式。
- 首次启动新版本前，为 `state.json` 自动创建备份。
- 新增设置必须提供默认值，旧 `state.json` 缺少字段时正常启动。
- v1.6.0 迁移 API Key 时，先读取安全存储，再兼容读取旧环境变量。

## 9. 风险与回滚

| 风险 | 控制措施 | 回滚方式 |
| --- | --- | --- |
| 自动保存频繁写盘 | 防抖、仅内容变化时保存 | 关闭自动保存开关，保留手动保存 |
| 钩子重构导致侧键失效 | 保留原回调语义，增加适配层测试 | 恢复上一版钩子实现 |
| 取消逻辑丢弃有效结果 | 使用单调递增 task_id 和明确状态机 | 临时关闭任务替换，仅允许单任务 |
| 原子保存兼容异常 | 同目录临时文件和备份文件 | 从 `.bak` 恢复原数据 |
| 安装器卸载规则误删 | 只删除固定名称的 Run 值 | 移除卸载 Registry 规则 |
| 图片兜底关闭降低成功率 | 提供明确的可选设置和 OCR 修复提示 | 用户主动开启图片兜底 |

## 11. 后续决策
